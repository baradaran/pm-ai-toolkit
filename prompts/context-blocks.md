# Context Blocks

Fill these in **once** for your product, then paste the relevant block at the start of any AI session. Half of bad AI output is just the model guessing context you could have given it. This fixes that.

Keep your filled-in versions in a private note (they may contain internal detail).

---

## Product context

```
PRODUCT CONTEXT
- Product: {{name + one-line what it is}}
- Stage: {{pre-PMF / growth / mature}}
- Business model: {{how we make money}}
- Primary users: {{role(s), what they're trying to do}}
- Buyer (if different): {{who pays}}
- Core value prop: {{the one promise}}
- Current strategic goal this cycle: {{e.g. activation, expansion, retention}}
```

## Product areas (your fixed taxonomy)

```
PRODUCT AREAS (use these exact labels for any theming/triage)
- {{Area 1, e.g. Onboarding}}
- {{Area 2, e.g. Billing}}
- {{Area 3, e.g. Search}}
- {{Area 4, e.g. Integrations}}
- Other
```
> Reuse the *same* labels every time — this is what makes feedback trends comparable across weeks.

## Audience profiles (for comms)

```
AUDIENCES
- Execs: care about {{outcomes, revenue, risk}}; allergic to {{jargon, activity reports}}.
- Engineering: care about {{clarity, trade-offs, flags}}.
- Sales/CS: care about {{what to say to customers, objection handling}}.
- Customers: care about {{benefits in plain language}}.
- {{Key account / stakeholder}}: hot buttons are {{...}}.
```

## Constraints

```
CONSTRAINTS
- Team: {{size / skills}}
- Timeline: {{}}
- Tech/platform limits: {{}}
- Compliance/data rules: {{what can't be shared or built}}
```

---

## Tips

- The **Product areas** block is the highest-leverage one — it powers [Feedback Triage](../tools/03-feedback-triage.md) and [Research Synthesizer](../tools/02-research-synthesizer.md) consistency.
- Update these blocks when strategy shifts, not per-session. They're meant to be stable.
- Never paste compliance-sensitive specifics into a tool you don't control.
